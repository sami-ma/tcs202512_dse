Creating Keyspace Replication 

NetworkTopologyStrategy : Used when Multi DC configuration is done
SimpleStrategy : Used when single DC configuration is done.


CREATE KEYSPACE [IF NOT EXISTS] keyspace_name
WITH REPLICATION = {
  'class' : 'NetworkTopologyStrategy',
  'DC1_name' : replication_factor_for_DC1 [, 'DC2_name' : replication_factor_for_DC2, ...]
}
[AND DURABLE_WRITES = true | false];


REPLICATION: Mandatory. 'class': 'NetworkTopologyStrategy' enables per-DC RF.

Datacenter options: Keys are exact DC names (case-sensitive); values are integers (e.g., 3 for 3 replicas in that DC).

DURABLE_WRITES: Defaults to true (recommended for production; commits to disk). Set to false only for performance in non-critical setups.

IF NOT EXISTS: Optional; skips creation if the keyspace already exists.


Examples:

CREATE KEYSPACE excalibur
WITH REPLICATION = {
  'class' : 'NetworkTopologyStrategy',
  'replication_factor' : 3
};



CREATE KEYSPACE myapp
WITH REPLICATION = {
  'class' : 'NetworkTopologyStrategy',
  'DC1' : 3,
  'DC2' : 2
}
AND DURABLE_WRITES = true;


CREATE KEYSPACE excalibur
WITH REPLICATION = {
  'class' : 'NetworkTopologyStrategy',
  'replication_factor' : 3,
  'DC2' : 2,
  'DC3' : 0
};
